# Gig Creation Guide — Web Scraping (Gig #1)

> Use this guide to manually create your Web Scraping gig on Fiverr.
> Each section maps directly to a field in Fiverr's gig creation form.

---

## 1. Start Creating a Gig

1. Log in to Fiverr (https://www.fiverr.com)
2. Click your profile icon → **Gigs** → **Create a New Gig**

---

## 2. Gig Title

**Max 80 characters**

```
I will scrape any website using Python and deliver clean data
```

*Character count: 65/80 — fits easily*

---

## 3. Category Selection Path

| Field | Selection |
|-------|-----------|
| **Category** | `Programming & Tech` |
| **Subcategory** | `Web Scraping` |
| **Detailed Category** | *(Leave blank — not required)* |

## 4. Tags

**Add exactly 5 tags** (click "+ Add" after each):

1. `web scraping`
2. `python scraping`
3. `data extraction`
4. `web crawling`
5. `scraping`

> ⚠️ If "python scraping" isn't available as a suggestion, try `python data scraping` or just `python`. Fiverr autocomplete may filter tags — pick the closest match from the dropdown.

## 5. Description

**Copy-paste the following into the gig description field.**

### Formatting Tip
Fiverr description supports basic Markdown. Paste the block below, then:
- Add line breaks between sections (use Enter)
- Bold the headings by selecting them and clicking **B** in the editor toolbar
- Or just paste as plain text — it will read well either way

---

**Paste this directly:**

```
Stop wasting hours manually copying and pasting data from websites. I build custom Python scrapers that extract exactly the information you need — clean, structured, and delivered on time.

Whether you need product listings, real estate data, job postings, news articles, business directories, or any other publicly available web data, I can build a reliable scraper that gets the job done.

Every scrape includes:
- Custom scraper built specifically for your target website(s)
- Anti-block techniques (rotating user agents, proxies, rate limiting)
- Data validation and error handling
- Clean, well-structured output files
- Free re-run if website structure changes within 30 days

My Process:

1. Discovery — You share the target URL(s) and what data you need. I assess complexity and provide a timeline.
2. Development — I build a Python scraper using requests, BeautifulSoup, Scrapy, Playwright, or Selenium depending on site complexity (static vs. JavaScript-rendered).
3. Testing — I run the scraper against a sample of your data, send you a preview, and make adjustments.
4. Delivery — You receive the full extracted dataset in your preferred format(s).
5. Support — I provide 7 days of post-delivery support for any questions or adjustments.

Why Choose Me:

- 5+ years of Python development — I've scraped hundreds of websites, from simple static pages to complex single-page applications and sites behind login walls.
- Clean, reliable code — Every scraper includes proper error handling, logging, and data validation. You won't get broken CSVs with missing rows.
- Anti-block expertise — I know how to avoid IP bans, CAPTCHAs, and rate limiting so your data arrives complete.
- Communication-first — I over-communicate throughout the project. No black box, no surprises.
- Post-delivery support — Websites change. If your target site updates its structure within 30 days, I'll re-run the scraper free of charge.

Note: I only scrape publicly available data. I do not scrape behind login walls, bypass paywalls, or violate any website's terms of service without explicit permission from the data owner.

Let's get started! Message me with your target URL and what data you need, and I'll provide a custom quote and delivery timeline within 24 hours.
```

## 6. Gallery Images

### Required: At least 3 images

Create these screenshots/graphics and upload them in the **Gallery** section:

| Image | What to Show | Suggested Content |
|-------|-------------|-------------------|
| **#1 — Gig Thumbnail / Main Image** | Text overlay: "Web Scraping — Clean Data from Any Website" on a tech-themed background | Python code on a dark background with the title. Use Canva template "Tech Services" or similar. |
| **#2 — Sample Output (CSV)** | Screenshot of the `cafe_listings.csv` file open in Excel/LibreOffice | Show the structured columns: Name, Address, Phone, Rating, Reviews. Reference file: `samples/cafe_listings.csv` |
| **#3 — Sample Output (JSON)** | Screenshot of structured JSON data | Show `cafe_listings` data in JSON format with clean nested fields. |
| **#4 — Process Diagram** | 3-step graphic: Discovery → Development → Delivery | Use simple icons/arrows. Canva "Process Flow" template works well. |
| **#5 — Deliverables Table** | Screenshot of the pricing/deliverables table | Capture the table from the description above, rendered nicely. |

**File specs for Fiverr:**
- Format: JPG, PNG, or GIF
- Max size: 8 MB per image
- Recommended resolution: 1280×769 px (or at least 640×384)
- At least 1 image required; 3+ recommended for higher conversion

## 8. Requirements (Questions for Buyer)

When a buyer orders, Fiverr will show these questions. **Add them in the "Requirements" section:**

> Click **+ Add Requirement** for each one.

| # | Question | Type | Required? |
|---|----------|------|-----------|
| 1 | What is the URL(s) of the website(s) you need scraped? | Short Answer | ✅ Required |
| 2 | What specific data fields do you need extracted? (e.g., name, price, address, rating, etc.) | Short Answer | ✅ Required |
| 3 | What format(s) do you want the data delivered in? (CSV, JSON, Excel, or all) | Multiple Choice | ✅ Required |
| 4 | How many pages or records do you need scraped? (approximate is fine) | Short Answer | ✅ Required |
| 5 | Does the website require login or authentication to access the data? | Yes/No | ✅ Required |
| 6 | Is this a one-time scrape or recurring (e.g., weekly, monthly)? | Multiple Choice | ✅ Required |
| 7 | Please share any reference/example of the data you're looking for (optional) | Attachment | Optional |
| 8 | Any specific deadlines or time constraints? | Short Answer | Optional |

### Multiple Choice Options

**Question 3 (Format):**
```
CSV only, JSON only, Excel only, CSV + JSON, All formats
```

**Question 6 (Recurring):**
```
One-time scrape, Weekly (every 7 days), Monthly, Custom schedule (we'll discuss)
```

### Suggested FAQs

> Click **+ Add FAQ** and fill in:

**Q: What types of websites can you scrape?**
A: Any publicly accessible website — product pages, directories, real estate listings, job boards, news sites, etc. JavaScript-heavy sites (React, Angular, Vue) may need a higher-tier package.

**Q: Do you offer revisions if I'm not satisfied?**
A: Yes! Each package includes revisions. I'll work with you until the data meets your requirements.

**Q: What if the website changes after delivery?**
A: If the site structure changes within 30 days of delivery, I'll re-run and update the scraper free of charge.

**Q: Can you scrape data behind a login?**
A: Yes, the Premium package includes login/authentication handling, provided you have legitimate access credentials and permission.
